## Decisions locked
- **Diagram orientation:** faithful/downward — high memory on top, `main()` oldest on top, newest frame at bottom = SP. Dual axis: **addresses increase upward, stack grows downward (opposite directions)** — a core teaching hook.
- **Diagram detail:** clean real-address boxes as default; **full frame anatomy** (all frames, params/return-addr/saved-FP/locals, per-slot byte size, per-row address) inside a `<details>`. Local address = **measured (red)**; other rows = **schematic (grey)**, computed by stacking `sizeof`-slots above the measured local.
- **Column ratio:** 2/3 code : 1/3 diagram (`code_diagram_panel` `2fr:1fr` for this subject).
- **Determinism contract:** stdout prints only deterministic facts (enter/leave traces, `sizeof`-derived slot-size summaries, memmap ordering booleans) → assertable; **raw addresses live only in `PTRDATA:` lines, drawn but never asserted**. Tests use substring checks (HTML-escaped, e.g. `text &lt; data? 1`).
- **Stepper = core (not deferred):** program emits one `PTRDATA` line per call/return; engine parses ALL lines (`parse_ptrdata_all`, additive — first-line `parse_ptrdata` untouched) → one SVG per step → CSS-radio control. Assumes each step's frame list is a prefix of the deepest (true for straight chains + recursion).
- **Dangling gotcha without ASan:** compile `sf_dangling_local` with **`-Werror=return-local-addr`** → red compile-error box (deterministic). Requires one additive engine field `extra_compile_flags` on `TopicTemplate` (Task 10).
- **`live` PTRDATA field format:** `name:addr:localname:localbytes:parambytes` (5th field optional). Consistent between the tracer and `_parse_frames`.
- **Execution mode:** **subagent-driven** (user directive), resuming after context clear.
