## Decisions / gotchas locked (also in project MEMORY.md)
- **SIA-R79 `<pre>` flag is a SiteImprove/Alfa quirk, NOT a defect.** User-tested: `language-cpp`
  (hljs highlights → spans) flags; `language-cppp` (hljs skips) doesn't; JS-off doesn't. So the trigger
  is the highlight.js span structure — its own "represents code ✓" passes. **Build-time baking would
  NOT clear it** (same spans). It's a best-practice badge, not a WCAG failure. Don't chase it.
- **JS stance relaxed:** "minimize JS, don't eliminate." Canvas is no longer the sole target, but pages
  should stay mostly functional there (JS-off = monochrome code, still readable). highlight.js (3 inline
  scripts) is acceptable.
- **Keyboard-nav help is for SIGHTED keyboard users** (SR users get the model from roles/names) → the
  help is VISIBLE, not sr-only.
- **`_ADDR_AXIS` green** and the active step/zoom button green are one value (`#2a7f54`); if changed,
  keep ≥4.5:1 on white AND with white text.
