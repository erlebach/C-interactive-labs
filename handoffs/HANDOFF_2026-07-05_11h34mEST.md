## Decisions / gotchas locked this session
- **Zero-JS trade-off (accepted):** the lightbox closes on click/✕/backdrop, **not ESC** (native `<dialog>` ESC needs scripting). The checkbox uses the clip pattern (not `display:none`) so it stays keyboard-focusable.
- **Chip-row gap:** the stylesheet `margin-right` was defeated by the chips' inline `margin` shorthand (inline beats stylesheet). Fixed by giving the glossary chip an inline **left-margin `.6rem`**; the dead stylesheet `margin-right` was dropped. (Final-review finding.)
- **Interface-catalog drift guard:** adding a `_BUILDERS` entry (`glossary_note`) **requires** matching rows in `interface_catalog.py`'s `_TIER` and `_BUILDER_INFO` — the generator KeyErrors otherwise. The per-task `cpp_labs/tests/` runs **exclude** `cpp_labs/yaml_engine/test_interface_catalog.py`, so this only surfaced at the Task 7 full-suite (`pytest cpp_labs`) gate. Fixed in a dedicated commit. **Lesson for future plans:** any `_BUILDERS`/`_DISPATCH` addition must include the catalog rows, and per-task verification should run `pytest cpp_labs -k interface_catalog` too.
