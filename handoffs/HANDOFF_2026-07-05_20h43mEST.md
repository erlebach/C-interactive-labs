## Decisions locked
- **Teach C++20 as the baseline; the axis is a small "standards awareness" subject, NOT sprinkled
  across every demo.** Rationale: tabs only teach when code is standard-sensitive; most topics compile
  identically across standards → three identical green tabs = noise. Existing subjects deliberately
  untouched. (See the mid-session discussion; validated the one-subject scope.)
- **Empirical verification is mandatory for every candidate feature.** Apple clang only *warns* (compiles
  green) on many "C++17/20" features — structured bindings, `string_view`, `if constexpr`, fold
  expressions, designated initializers, variable templates, inline static vars. These are **unusable**
  for red/green. Also `std::bit_cast` and `std::ranges` are **unsupported even at C++20** on this
  libc++. Only use features proven to hard-error under the old `-std` (test in `/tmp` before authoring).
- **Standards-only rule:** a `standards:` topic may not also have a dropdown control (one tab row only);
  duplicates rejected. Cartesian combos deferred.
- **`__std__`** is a synthetic control-state key — string values, never substituted into templates
  (`generate_source` iterates only `topic.controls`).
- **Label taxonomy:** demonstration objective = **Main Takeaway** (sidebar `concept_panel`); per-example
  = **Key Idea** (both `concept_note` toggle and `code_concept_panel` aside).
